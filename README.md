# 术图 Shutu Analysis

术图是一个面向中文学术场景的数据可视化分析助手。它将结构化数据输入转化为可阅读的分析报告与三套互补图表方案，并提供可交互渲染、可复制复现代码、可分享和可导出能力。

## 核心能力

- 双阶段分析链路：先生成数据概览，再生成三套创新视角方案，结果以流式方式展示。
- 支持普通模式与高级模式：
  - 普通模式：无 Key 也可运行，返回内置规则模板结果。
  - 高级模式：接入 DeepSeek，生成高质量长文本与方案代码。
- 支持多种数据输入：
  - 拖拽上传 `CSV`、`XLSX`、`XLS`
  - 直接粘贴表格文本
  - 通过公开 Google Sheets 链接导入
- 自动字段推断与推荐图表：
  - 自动识别数值/分类/时间字段
  - 自动推荐图表类型
  - 提供概览图参数面板，可手动切换图表类型与字段
- 三方案展示与编辑：
  - 每套方案包含说明、Plotly JSON、Python 代码
  - 内置 Plotly JSON 编辑器，支持实时应用重新渲染
  - 代码块支持一键复制
- 通俗解读增强：
  - 可调用 ECNU 接口对当前激活方案生成非技术语言解读
- 结果管理：
  - 历史记录（本地存储）
  - 导出 Markdown 文本、长图 PNG、A4 纵向 PDF
  - 生成 24 小时有效的只读分享链接
- 只读分享页：
  - 独立 `/s/:id` 路由
  - 可查看图表与方案内容，不包含编辑与原始输入

## 功能全景

### 1) 输入与预设面板

- 快速模板：描述统计、相关性分析、分组对比、时间趋势、异常与分布结构。
- 具体需求输入：可补充分析目标、报告风格、结论偏好。
- 原始数据输入：
  - 上传文件后自动解析并同步到文本区
  - 手动改文本后自动重算字段与推荐
  - 支持 Google 表格公开链接转 CSV
- 字段与推荐展示：
  - 展示已识别字段类型
  - 展示推荐图表类型及原因
- 中文版式与参数预设：
  - 字体、字号、配色、DPI、宽度参数可选
  - 参数会写入提示词，影响生成风格

### 2) 分析与生成

- 生成按钮触发后进入流式状态，状态分为：
  - 生成数据概览
  - 撰写三套方案
  - 图表通俗解读
- 解析规则基于固定分隔符：
  - `===方案一===`
  - `===方案二===`
  - `===方案三===`
- 每套方案支持：
  - Plotly 图表渲染
  - Python 复现代码展示与复制
  - 方案说明 Markdown 渲染

### 3) 结果区操作

- 方案卡片切换与当前卡解读
- 导出文本、长图、PDF
- 生成只读分享链接并自动复制
- 二次修改入口：
  - 输入修改要求后仅重写方案部分
  - 不重复完整概览

### 4) 历史记录

- 自动保存最近分析结果（本地）
- 支持加载历史条目恢复到工作区
- 支持清空历史

### 5) 分享系统

- 后端生成短 ID，并在内存中保存快照
- 默认有效期 24 小时
- 分享页只读，不含编辑器与输入区域

## 技术架构

### 前端

- `React 19 + TypeScript + Vite`
- 图表渲染：`plotly.js + react-plotly.js`
- Markdown：`react-markdown + remark-gfm`
- 代码高亮：`react-syntax-highlighter`
- 导出：`html2canvas + jspdf`

### 后端

- `Express + TypeScript (tsx watch)`
- 能力：
  - DeepSeek 流式代理
  - ECNU 聊天接口转发
  - Google Sheets 抓取转 CSV
  - 分享快照存取
  - 千帆图像生成接口（服务端能力已具备）

### 运行端口

- 前端：`http://localhost:3000`
- 后端：`http://localhost:3001`
- 前端通过 Vite 代理 `/api/*` 到后端

## 安装与启动

### 1) 安装依赖

```bash
npm install
```

### 2) 配置环境变量

复制 `.env.example` 为 `.env`，按需填写：

```env
GEMINI_API_KEY=""
APP_URL=""
DEEPSEEK_API_KEY=""
DEEPSEEK_BASE_URL="https://api.deepseek.com"
QIANFAN_API_KEY=""
QIANFAN_SECRET_KEY=""
```

说明：

- `DEEPSEEK_API_KEY`：高级模式下可作为后端默认 Key。
- `DEEPSEEK_BASE_URL`：DeepSeek OpenAI 兼容基地址。
- `QIANFAN_*`：图片生成接口使用；如仅使用图表分析，可先留空。
- `GEMINI_API_KEY`、`APP_URL`：保留兼容字段，不是当前主流程必需项。

### 3) 启动开发环境

```bash
npm run dev
```

该命令会同时启动前端与后端服务。

### 4) 其他命令

```bash
npm run dev:vite
npm run dev:server
npm run build
npm run preview
npm run lint
```

## 使用手册

### 第一步：进入数据可视化工作区

- 打开首页后选择「数据可视化分析」进入主界面。

### 第二步：输入数据

- 推荐优先使用上传：
  - CSV、XLSX、XLS
- 或直接粘贴表格文本（支持逗号/分号/Tab 分隔）。
- 或使用公开 Google Sheets 链接导入。

### 第三步：设置分析需求

- 先选快速模板，再补充具体需求。
- 可在「中文版式与参数预设」中设置字体、配色、DPI、宽度等参数。

### 第四步：配置 API（可选但推荐）

- 打开「API 设置」：
  - 可填 ECNU API Key，用于数据概览和图表通俗解读
  - 可填 DeepSeek API Key，用于高级模式下方案生成
  - 可切换 `deepseek-chat` 或 `deepseek-reasoner`

### 第五步：开始分析

- 点击「开始智能分析」。
- 系统将流式生成：
  - 数据参考分析报告
  - 三套方案（每套含图表 JSON 与 Python 代码）

### 第六步：编辑与导出

- 在方案卡片中查看渲染图与说明。
- 使用 Plotly JSON 编辑器修改并应用。
- 导出文本、长图、PDF，或创建分享链接。

### 第七步：二次修改

- 在「对当前结果不满意？」输入修改要求。
- 系统会重写方案部分，保留整体上下文。

## 配置说明与策略

### 模式策略

- 普通模式：
  - 不依赖外部大模型 Key
  - 返回规则模板分析结果
- 高级模式：
  - 优先使用用户在界面填写的 DeepSeek Key
  - 若用户未填，尝试使用后端环境变量 `DEEPSEEK_API_KEY`
  - 两者都缺失时回退普通模式

### ECNU 分流策略

- 若提供 ECNU Key：
  - 数据概览由 ECNU 生成
  - 三套方案由 DeepSeek 生成
- 若未提供 ECNU Key：
  - 全流程由 DeepSeek 或普通模式回退完成

### 本地存储项

- `shutu_deepseek_api_key`
- `shutu_ecnu_api_key`
- `shutu_deepseek_model`
- `shutu_analysis_mode`
- `shutu_history`

## 后端接口一览

### Chat 相关

- `POST /api/chat/session`：创建流式会话
- `GET /api/chat?sessionId=...`：SSE 拉取 token 流
- `POST /api/ai/deepseek/generate`：纯文本流式代理
- `POST /api/ecnu/chat`：ECNU 接口转发

### 数据与分享

- `POST /api/sheets/fetch`：公开 Google Sheets 转 CSV
- `POST /api/share`：创建分享快照
- `GET /api/share/:id`：读取分享快照

### 图像

- `POST /api/images/generate`：千帆文生图

## 重要注意事项

### 安全与密钥

- 不要把 `.env` 提交到仓库。
- 不要在公开场合暴露 API Key。
- 前端 API 设置里的 Key 存于浏览器 `localStorage`，请在公共电脑使用后手动清理。

### 数据与隐私

- 原始数据主要在本地前端处理和预分析。
- 当调用 ECNU/DeepSeek 时，系统会发送生成所需文本内容到对应服务。
- 分享链接是只读快照，不包含输入面板和编辑动作。

### 分享机制限制

- 分享数据默认仅内存保存 24 小时。
- 服务重启后内存快照会丢失，即使未到 24 小时也可能失效。

### Google Sheets 导入限制

- 链接必须是公开可读。
- 必须包含 `/spreadsheets/d/...`。
- 单次抓取上限约 8MB，过大请改用文件上传。

### 数据规模与性能建议

- 上传文件解析后会做行数截断以保证交互性能。
- 数据量很大时建议先采样或聚合后再分析。
- 大量复杂图表会增加浏览器内存与导出耗时。

## 常见问题

### 1) 点击分析后提示 Key 问题

- 检查 API 设置中是否填写正确的 ECNU/DeepSeek Key。
- 检查 `.env` 中后端默认 Key 是否可用。

### 2) Google 表格导入失败

- 确认表格共享为「知道链接的任何人可查看」。
- 确认链接完整且包含工作表 `gid`。

### 3) SSE 流中断

- 检查后端服务是否正常运行在 `3001`。
- 检查代理配置和网络连接。

### 4) 分享链接打不开

- 可能已过期或服务重启导致内存快照丢失。
- 重新生成分享链接即可。

### 5) 导出图片或 PDF 效果异常

- 等待图表完全渲染后再导出。
- 避免一次性渲染过多超大图表。

## 生产部署建议

- 使用 `npm run build` 生成前端静态资源。
- 后端建议使用 `pm2`、`systemd` 或容器化托管。

- 通过 `VITE_PROXY_TARGET` 配置反向代理目标。
